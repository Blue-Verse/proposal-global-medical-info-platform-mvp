핵심 요구사항 분석

  • 완전 독립형 자체 구축 — 임대형·기성 솔루션 불가, 100% 자체 소스코드로 프레임워크부터 구축
  • 한국어·일본어 다국어 — 언어 선택 및 전체 화면 다국어 전환, KR/JP 동시 서비스
  • 사용자(환자) 기능 — 성형외과·치과 카테고리별 병원 목록·상세 조회, 후기 열람, 방문 날짜·시간 지정 상담 예약 신청
  • 입점 병원 기능 — 병원 계정 로그인, 예약 접수 내역 확인, 병원 정보·시술·후기 수정 요청
  • 최고 관리자 기능 — 병원 계정 생성·권한 부여, 병원 정보·시술·후기 대리 입력 및 발행 승인, 전체 예약 통합 조회·상태 변경, 회원 조회·차단
  • KR/JP 인프라 — 한국·일본 접속 속도를 고려한 클라우드 서버 및 CDN 구성
  • 디자인 최우선 — 레퍼런스 기반 하이엔드 플랫폼 UI/UX 설계, 디자인 완성도가 1순위 평가 기준
  • 범위 제외(1차) — 온라인 결제 및 수수료 정산 기능은 MVP 범위에서 제외

제안 포인트

1. 디자인 완성도 최우선 접근. 본 프로젝트가 UI/UX 디자인을 최우선 평가 기준으로 명시한 만큼, 레퍼런스 분석 → 디자인 시스템 정의 → 하이엔드 시안 → 개발의 순서로 진행하며 디자인 원본 파일(Figma)을 산출물로 제공합니다. 의료 정보 서비스 특성상 신뢰감과 가독성을 우선한 톤앤매너를 적용합니다.

2. 다국어를 데이터·설계 단계부터 내장. 화면 텍스트 번역 수준이 아니라, 콘텐츠(병원 소개·시술·후기)의 언어별 필드 구조와 i18n 키 관리 정책을 기획 단계에서 확정하여 한국어·일본어 전체 전환이 자연스럽게 동작하도록 설계합니다. 유사하게 2개 언어·수천 개 번역 키를 관리한 다국어 플랫폼 경험을 보유하고 있습니다.

3. 3역할 권한 구조와 발행 승인 워크플로우. 사용자(환자)·입점 병원·최고 관리자 3역할의 권한 경계를 RBAC로 명확히 분리하고, 병원이 요청한 정보를 최고 관리자가 대리 입력·검수·발행 승인하는 워크플로우를 구현합니다. 다단계 승인·발행 워크플로우와 셀프서비스 포털을 다수 구축한 경험을 그대로 적용합니다.

4. KR/JP 이원 인프라로 접속 속도 확보. AWS 서울(ap-northeast-2)·도쿄(ap-northeast-1) 리전과 CloudFront CDN을 활용하여 양국 사용자 모두 빠른 응답 속도를 확보합니다. 인프라를 코드(IaC)로 관리하여 이관·확장이 용이합니다.

5. 확장을 염두에 둔 MVP. 1차에서 제외된 결제·정산 기능을 이후 손쉽게 추가할 수 있도록, 예약·정산 도메인의 확장 지점을 미리 설계에 반영하여 시장 검증 후 빠른 고도화가 가능하도록 합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

기존 쇼피파이·카페24 기반 솔루션은 글로벌 확장에 한계가 있고 국내 사용자 환경에 부적합합니다. 이를 해결하기 위해 완전 독립형 자체 구축 플랫폼을 개발하며, 복잡한 온라인 결제·수수료 정산은 1차 범위에서 제외하고 병원 정보 제공과 사용자 예약 접수에 집중하여 빠른 시장 검증을 목표로 합니다. 한국·일본 두 시장을 동시에 타겟합니다.

2. 사용자 정의

사용자 유형설명주요 기능
사용자(환자)성형외과·치과 정보를 탐색하고 예약하는 일반 이용자언어 선택, 카테고리별 병원 목록·상세·후기 열람, 방문 날짜·시간 지정 상담 예약 신청
입점 병원플랫폼에 입점한 개별 병원 계정로그인, 예약 접수 내역 확인, 병원 정보·시술·후기 수정 요청
최고 관리자플랫폼 운영 주체병원 계정 생성·권한 부여, 정보·시술·후기 대리 입력 및 발행 승인, 전체 예약 통합 조회·상태 변경, 회원 조회·차단

3. 기능 요구사항

3.1 다국어

  • [필수] 한국어·일본어 언어 선택 및 전체 화면 다국어 전환
  • [필수] 병원 소개·시술·후기 등 콘텐츠의 언어별 관리 구조

3.2 사용자(환자) 기능

  • [필수] 성형외과·치과 카테고리별 입점 병원 목록 조회
  • [필수] 병원별 상세 페이지 — 병원 소개·시술 정보·사용자 후기 출력
  • [필수] 방문 날짜·시간 선택을 통한 병원 상담 예약 신청

3.3 입점 병원 기능

  • [필수] 개별 병원 계정 로그인 및 예약 접수 내역 확인
  • [필수] 병원 정보·시술 내역·후기 수정 사항의 최고 관리자 요청

3.4 최고 관리자 기능

  • [필수] 입점 병원 신규 계정 생성 및 관리자 권한 부여
  • [필수] 병원 상세 정보·시술·후기 대리 입력 및 발행 승인
  • [필수] 전체 사용자 예약 접수 내역 통합 조회 및 상태 변경
  • [필수] 가입 회원 정보 조회 및 차단 관리

5. 기술 제약사항

  • 임대형·기성 솔루션(워드프레스·카페24·쇼피파이) 사용 불가 — 100% 자체 소스코드 구축 필수

프로젝트 개요

종사자·어르신·보호자를 연결하는 헬스케어 통합 관리 시스템. 돌봄 기록·투약·보호자 소통·AI 건강분석을 하나의 플랫폼으로 통합하고, 관리자는 웹 패널로 다중 시설을 운영합니다.

핵심 기능

  • 2개 언어·2,532 다국어 키 관리, 6개 플랫폼 지원

본 프로젝트와의 유사점

병원/시설을 테넌트로 관리하는 구조, 관리자 웹 패널을 통한 정보 관리·발행, 헬스케어 도메인 특유의 신뢰성 요구, 다국어 관리 경험이 본 프로젝트의 입점 병원·최고 관리자 구조와 직접 맞닿아 있습니다.
